üßÆ What is Map Algebra?

Raster analysis is fundamentally about mathematics. Because raster data is stored as a matrix of numbers (pixels), we can perform mathematical operations on them. This concept was formalized by Dana Tomlin as Map Algebra.

The core idea is simple:
Input Layer(s) + Operation = Output Layer

üìê The Four Types of Operations

Raster functions are categorized by the scope of pixels they consider when calculating the value for a single output cell.

üìç

Local

Cell-by-Cell

The value depends ONLY on the cell at the same location in input layers.
Ex: Map Algebra (A + B), Reclassification

üîç

Focal

Neighborhood

The value depends on the cell plus its immediate neighbors (e.g., 3x3 window).
Ex: Smoothing, Edge Detection, Slope

üó∫Ô∏è

Zonal

Region/Zone

The value depends on all cells that share the same "zone" value.
Ex: Average Elevation per County, Zonal Statistics

üåê

Global

Whole Map

The value depends on potentially ALL cells in the raster.
Ex: Euclidean Distance, Cost Path, Watersheds

ü¶Å Case Study: Habitat Suitability

The most common application of raster analysis is Suitability Modeling (or Site Selection). We define criteria for an ideal location (e.g., "Flat land" AND "Near Water" AND "Far from Roads") and combine layers to find the perfect spot.

Binary vs. Weighted Overlay

  • Binary (Boolean): 1 (Suitable) or 0 (Not Suitable). Like a hard filter.
  • Weighted: Scale of 1-10. Slope matters 50%, Water matters 30%. Allows for "better" vs "worse" sites.
