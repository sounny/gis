In a GIS, geometry is only half the story. The Attribute Table contains the actual information about those features. These tables are organized using a Relational Database Management System (RDBMS).

üîë The Primary Key: Every row in a GIS table must have a unique identifier (often called an FID or OBJECTID). This allows the software to link a geometric shape on the map to a specific row in the database.

One of the most powerful tasks in GIS is a Join. By matching a common field (the "foreign key") in two different tables, we can temporarily append information from a standalone table to our spatial features.

Example: Joining 2024 Census Data (table) to County Boundaries (spatial layer) using the "FIPS Code" as the link.

Summary of Big Ideas

  • Attribute Tables store the "what" for every "where."
  • Primary Keys ensure every feature is unique and reachable.
  • Table Joins allow external data to be visualized spatially.
  • SQL is the universal language for querying geographic databases.

Local to Global

The Texas RRC Well Database

The Texas Railroad Commission (RRC) manages one of the largest spatial databases in the world. Every oil and gas well in Texas is assigned a unique API Number. By using this primary key, engineers can join production reports, depth logs, and ownership records to a shared coordinate on a map.

This integration is what makes Texas a global leader in energy data management.
